STEP 3: Data Modeling (connecting tables)


🔗 STEP 3: CONNECT ALL 3 TABLES (DATA MODELING)
🧠 First Understand the Goal

You have 3 tables:

Sales
Product_Cost
Targets

👉 We need to connect them like this:

Product_Cost → Sales ← Targets

🟢 Step 1: Go to Model View

In Power BI (left side), click:

Model icon (diagram symbol)

👉 You’ll see all 3 tables

🟢 Step 2: Connect Product_Cost → Sales
🎯 Goal:

Connect products with their cost

📌 Do this:

In Sales table, find:

Product
Click and HOLD it

Drag it to:

Product_Cost → Product
Release mouse
⚙️ A popup may appear:

Set:

Cardinality:

Many to One (*:1)
Click:
OK
✅ Result:

You’ll see a line:

Sales (*) -------- (1) Product_Cost

🟢 Step 3: Connect Targets → Sales (IMPORTANT)
🎯 Goal:

Connect Month with target

⚠️ Important Check First:

Make sure:

Sales has Month column (Jan, Feb, Mar)
Targets has Month column
📌 Do this:

In Sales table, find:

Month

Drag it to:

Targets → Month
⚙️ Set:
Cardinality:
Many to One (*:1)

Click:

OK
✅ Result:
Sales (*) -------- (1) Targets

🟢 Step 4: Final Model Should Look Like
        Product_Cost
             |
             |
           Sales
             |
             |
          Targets

👉 This is again a Star Schema ⭐

🟢 Step 5: Verify Relationships

Click each line and check:

* -------- 1

👉 Means:

Sales = Many side
Others = One side

🟢 Step 6: Test If It Works

Go to Report View

Test 1:
Create a table visualization

Add:
Product (from Product_Cost)
Sales (from Sales)

👉 If data appears → ✅ working

Test 2:
Add:
Month (from Sales)
Target_Sales (from Targets)

👉 If values show → ✅ working

⚠️ Common Mistakes (Very Important)
❌ Relationship not working?

Check:

Column names must match exactly
Month format same (Jan vs January ❌)
❌ Wrong Cardinality?

👉 Always:

Many (Sales) → One (lookup tables)
❌ Duplicate values in lookup table?

👉 Remove duplicates in:

Product_Cost
Targets
💡 Pro Tip (Important)

Think like this:

Sales = main data (transactions)
Others = supporting tables
