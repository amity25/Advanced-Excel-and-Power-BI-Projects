DATA MODELING IN POWER BI

👉 This is the first real “advanced” skill in Power BI
👉 Most freshers skip this—so this gives you an edge

🧠 What You’ll Learn in This Step

Right now your data is in one single table ❌
We’ll convert it into a proper data model (multiple tables + relationships) ✅

This is how real companies work.



🟢 Step 1: Understand the Goal

We will split your data into 3 tables:

📊 Final Structure:
Cleaned Table (Fact Table)
→ Contains transactions (Sales, Quantity, Revenue)
Customer Table (Dimension)
→ Customer details
Product Table (Dimension)
→ Product & Category info


🟢 Step 2: Duplicate Your Table (Important)
Open Power BI

Go to:

Home → Transform Data

(This opens Power Query)

Now:

Right-click your table → Click:

Duplicate

👉 Do this 2 times

Now you have:

Cleaned (original)
Cleaned (2)
Cleaned (3)


🟢 Step 3: Create Customer Table
Click Cleaned (2)

Rename it:

Customers
Keep only these columns:
Customer_Name
Region
Remove others:
Select unwanted columns
Right-click → Remove Columns
Remove duplicates:
Select both columns

Click:

Remove Rows → Remove Duplicates

👉 Now you have a clean Customer table



🟢 Step 4: Create Product Table
Click Cleaned (3)

Rename it:

Products
Keep only:
Product
Category
Remove duplicates (same way)

👉 Now you have a Product table



🟢 Step 5: Load Data

Click:

Close & Apply


🟢 Step 6: Create Relationships (MOST IMPORTANT)
Go to Model View (left side → diagram icon)

Now you’ll see all tables

Create relationships:

🔗 A. Customer Relationship

Drag:

Cleaned[Customer_Name] → Customers[Customer_Name]

🔗 B. Product Relationship

Drag:

Cleaned[Product] → Products[Product]

👉 Relationships should be:

One-to-Many (* → 1)


🟢 Step 7: Verify Model

You should now see:

Customers → Sales → Products

👉 This is called a Star Schema ⭐
